## 第一部分：基线测试（现有设计）

### 脚本说明
**时长**：约15分钟
**目的**：收集现有设计的基线数据
**指导语**：
> "接下来请您完成几个任务。请尽量独立完成，如果实在无法完成可以告诉我。完成任务的过程中，我会记录您的时间和操作。有任何问题随时告诉我。"

### 任务 T1.1：身份选择
**场景**：您第一次打开这个小程序
**目标**：选择您的身份（健听人/听障人士）

**指导语**：
> "假设这是您第一次使用这个小程序，请根据您的实际情况选择身份。"

**观察要点**：
- [ ] 是否注意到身份选择
- [ ] 选择过程是否犹豫
- [ ] 是否正确进入对应模式
- [ ] 是否有误操作

**数据记录**：
```

开始时间: ____
完成时间: ____
是否成功: 是 / 否
帮助次数: ____
错误操作: ________________

```

## 第二部分：改进设计测试

### 过渡说明
**指导语**：
> "接下来我要向您介绍一个新版本。这个版本针对刚才的使用体验做了一些改进。让我先简单介绍一下新功能..."

### 新功能介绍（5分钟）

#### 1. 高对比度模式
**演示**：
> "这是高对比度模式，文字和背景的对比更强，更容易看清。"

**让用户尝试**：
- [ ] 开启/关闭高对比度
- [ ] 在不同页面查看效果

#### 2. 字体缩放
**演示**：
> "您可以根据需要调整字体大小，从标准到最大有三级。"

**让用户尝试**：
- [ ] 调整到150%
- [ ] 调整到200%
- [ ] 观察布局变化

#### 3. 触觉反馈
**演示**：
> "现在点击按钮时会有轻微的震动反馈，让您知道操作已被接收。"

**让用户尝试**：
- [ ] 点击不同按钮感受反馈差异
- [ ] 调整震动强度

#### 4. 减少动画
**演示**：
> "如果您对动画敏感，可以关闭或减少动画效果。"

**让用户尝试**：
- [ ] 开启/关闭动画
- [ ] 对比体验

---

### 练习时间（5分钟）
**指导语**：

- [ ] 用户首先尝试哪些功能

6. 我认为有太多不一致的地方。                      [5] [4] [3] [2] [1] ←反向
7. 我认为大多数人会很快学会使用。                  [1] [2] [3] [4] [5]
8. 我发现这个 cumbersome 难用。                   [5] [4] [3] [2] [1] ←反向

## 常见问题应对

### 用户无法完成任务
**应对**：
> "没关系，这正好帮助我们发现问题。您可以告诉我您想怎么做，或者需要什么帮助。"

**记录**：
- 用户尝试的方法
- 卡点位置
- 用户的期望

### 用户情绪焦躁
**应对**：
> "这只是一个测试，没有对错之分。如果您需要休息，我们可以暂停一下。"

**措施**：
- 提供水
- 允许休息
- 必要时中止测试

### 设备/技术问题
**应对**：
- 立即切换到备用设备
- 记录问题详情
- 调整测试计划

### 用户要求帮助
**原则**：
- 首次请求：鼓励再尝试
- 第二次：提供最小提示
- 第三次：直接帮助并记录
