# "心语者"小程序无障碍设计对比实验方案

## 一、实验概述

### 1.1 实验背景
"心语者"是一款面向听障人士与健听人群的手语翻译小程序，现有设计存在以下无障碍问题：
- 色彩对比度不足（如次要文字 #666666 在 #FFFFFF 背景上对比度仅 5.74:1，勉强达标）
- 缺少高对比度模式支持
- 字体大小固定，未适配系统字体缩放
- 缺少触觉反馈机制
- 动画效果无法根据用户偏好关闭
- 屏幕阅读器支持不完善

### 1.2 实验目标
1. **对比现有设计 vs 改进设计的无障碍表现**
   - 量化WCAG 2.2 AA合规性提升程度
   - 识别关键改进点与剩余问题

2. **验证WCAG合规性改进的实际效果**
   - 任务完成时间减少比例
   - 错误率降低程度
   - 用户满意度提升

3. **测试不同辅助功能的需求程度**
   - 各用户群体对特定功能的依赖度
   - 功能优先级排序依据

### 1.3 实验设计类型
**混合方法设计**：
- 定量：A/B测试 + 任务完成度量
- 定性：访谈 + 认知走查

## 二、实验维度详解

### 2.1 维度一：色彩对比度

#### 测试条件
| 条件 | 描述 | WCAG目标 |
|------|------|----------|
| A-现有设计 | 当前配色方案 | 部分达标 |
| B-高对比度模式 | 黑底白字 + 高饱和强调色 | 完全达标 |
| C-用户自定义 | 可调节对比度滑块 | 个性化达标 |

#### 具体参数
```css
/* 现有设计 */
--text: #1A1A2E;           /* 对比度 ~15:1 达标 */
--text-secondary: #666666; /* 对比度 ~5.74:1 边缘 */
--text-muted: #999999;     /* 对比度 ~2.8:1 不达标 */

/* 高对比度模式 */
--hc-text: #FFFFFF;        /* 对比度 21:1 */
--hc-bg: #000000;
--hc-accent: #FFFF00;      /* 对比度 19:1 */
```

#### 测试指标
- 文字识别准确率
- 界面元素定位时间
- 视觉疲劳评分（1-10量表）

### 2.2 维度二：字体缩放

#### 测试条件
| 缩放级别 | 基准字体大小 | 适用场景 |
|----------|--------------|----------|
| 100% | 28rpx (14px) | 标准视力 |
| 150% | 42rpx (21px) | 轻度低视力 |
| 200% | 56rpx (28px) | 中度低视力 |

#### 技术实现要点
```css
/* 响应式字体 */
.font-scalable {
  font-size: calc(var(--base-size) * var(--user-scale));
}

/* 容器适配 */
.container-fluid {
  min-height: calc(100vh / var(--user-scale));
}
```

#### 测试指标
- 阅读速度（字/分钟）
- 内容截断/重叠问题数
- 布局错乱发生率

### 2.3 维度三：触觉反馈

#### 测试条件
| 模式 | 震动强度 | 触发场景 |
|------|----------|----------|
| 无震动 | 0ms | 对照组 |
| Light | 10ms | 按钮点击 |
| Medium | 30ms | 操作成功 |
| Heavy | 50ms | 紧急/错误提示 |

#### 触发场景映射
```javascript
const hapticPatterns = {
  'button-press': { intensity: 'light', duration: 10 },
  'action-success': { intensity: 'medium', duration: 30 },
  'action-error': { intensity: 'heavy', duration: 50 },
  'urgent-alert': { intensity: 'heavy', duration: 100, repeat: 2 },
  'progress-tick': { intensity: 'light', duration: 5 }
};
```

#### 测试指标
- 操作确认感知率
- 错误响应时间
- 用户主观偏好

### 2.4 维度四：视觉反馈

#### 测试条件
| 反馈类型 | 描述 | WCAG关联 |
|----------|------|----------|
| 简单变化 | 颜色/透明度变化 | 2.3.1 闪烁限制 |
| 波纹动画 | Material Design风格 | 2.2.2 可暂停 |
| 颜色闪烁 | 高对比度闪烁 | 2.3.1 禁止 |
| 边框强调 | 轮廓线变化 | 2.4.7 焦点可见 |

#### 动画参数
```css
/* 符合WCAG的动画 */
--duration-fast: 150ms;    /* 认知负荷低 */
--duration-normal: 200ms;
--duration-slow: 300ms;    /* 感知清晰 */

/* 减少动画偏好 */
@media (prefers-reduced-motion: reduce) {
  * { animation-duration: 0.01ms !important; }
}
```

## 三、测试用户招募

### 3.1 用户分组

#### 组A：低视力用户（n=8-10）
- **纳入标准**：
  - 最佳矫正视力 0.1-0.3 或视野 < 20°
  - 日常使用屏幕放大器或高对比度设置
  - 年龄 18-70 岁
- **排除标准**：
  - 完全失明（归入组D）
  - 认知障碍影响任务理解

#### 组B：老年用户（n=8-10）
- **纳入标准**：
  - 年龄 ≥ 65 岁
  - 自报有老花眼或轻度视力退化
  - 非日常小程序重度用户
- **排除标准**：
  - 严重视力障碍（归入组A）
  - 严重听力障碍（归入组C）

#### 组C：手语优先型听障用户（n=8-10）
- **纳入标准**：
  - 手语为主要沟通方式
  - 听力损失 > 70dB 或自幼失聪
  - 使用辅助技术（如振动提醒）
- **排除标准**：
  - 口语为主要沟通方式

#### 组D：辅助技术使用者（n=6-8）
- **纳入标准**：
  - 日常使用屏幕阅读器（VoiceOver/TalkBack）
  - 或日常使用屏幕放大器
  - 或同时使用多种辅助技术
- **子分组**：
  - D1：全盲用户（屏幕阅读器）
  - D2：低视力用户（放大器+阅读器）

### 3.2 招募渠道
| 渠道 | 目标组 | 预期招募数 |
|------|--------|-----------|
| 残联合作机构 | A, C, D | 15 |
| 老年大学/社区 | B | 10 |
| 无障碍技术社群 | D | 5 |
| 听障学校/协会 | C | 5 |

### 3.3 补偿标准
- 时长：90-120分钟
- 补偿：200元/人 + 交通补贴
- 额外：提供无障碍改进版小程序优先体验权

### 4.2 任务二：紧急功能定位速度

**场景B**：模拟医疗场景 - "你需要表达身体不适"
- 目标：找到并触发"我不舒服"短语
- 测量：同上

- 首次注视时间（Time to First Fixation）
- 注视点数量
- 扫描路径效率

#### 测量指标
| 指标 | 测量方式 | 时间点 |
|------|----------|--------|
| 主观疲劳 | NASA-TLX量表 | 阅读前/后 |

| 眨眼频率 | 眼动仪（可选） | 每5分钟 |

#### 错误场景设计
| 场景ID | 错误类型 | 触发方式 |
|--------|----------|----------|
| E1 | 误触删除 | 长按删除常用语时误操作 |
| E2 | 网络中断 | 模拟弱网环境 |
| E3 | 输入错误 | 超长文本输入 |
| E4 | 权限拒绝 | 相机权限未授予 |

#### 恢复指标
- 错误发现时间
- 恢复操作时间
- 是否需要外部帮助
- 恢复后是否继续任务

#### 效应量计算
```python
# Cohen's d
def cohens_d(x, y):
    nx = len(x)
    ny = len(y)
    dof = nx + ny - 2
    pooled_std = np.sqrt(((nx-1)*x.var() + (ny-1)*y.var()) / dof)
    return (x.mean() - y.mean()) / pooled_std

# 解释：
# d = 0.2 小效应
# d = 0.5 中效应
# d = 0.8 大效应
```
