# "心语者"小程序无障碍设计对比实验

## 项目概述

本项目为"心语者"手语翻译小程序设计了一套完整的无障碍对比实验方案，旨在：

1. 对比现有设计与改进设计的无障碍表现
2. 验证WCAG 2.2 AA合规性改进的实际效果
3. 测试不同辅助功能的需求程度
4. 为产品优化提供数据支撑

## 核心内容概览

### 1. 实验设计

**实验类型**：混合方法设计（定量A/B测试 + 定性访谈）

**实验维度**：
- 色彩对比度（现有 vs 高对比度）
- 字体缩放（100% vs 150% vs 200%）
- 触觉反馈（无 vs light vs medium vs heavy）
- 视觉反馈（简单 vs 波纹动画 vs 减少动画）

**测试用户组**：
- 低视力用户（n=8-10）
- 老年用户 65岁以上（n=8-10）
- 手语优先型听障用户（n=8-10）
- 辅助技术使用者（n=6-8）

### 2. WCAG 2.2 AA 合规性检查

检查清单涵盖：
- 可感知性（文本替代、色彩对比度、字体缩放等）
- 可操作性（键盘访问、焦点可见、目标尺寸等）
- 可理解性（错误识别、标签说明等）
- 健壮性（兼容性、状态消息等）

**当前发现的主要问题**：
- 弱化文字对比度不足（2.84:1，需≥4.5:1）
- 占位符文字对比度不足
- 禁用状态对比度不足
- 输入框边框对比度不足
- 目标尺寸需验证是否符合24×24px要求

### 3. 测试任务

| 任务ID | 描述 | 测量指标 |
|--------|------|----------|
| T1.1 | 身份选择 | 完成时间、成功率 |
| T1.2 | 文字转手语 | 完成时间、输入效率 |
| T1.3 | 使用快捷短语 | 定位时间、反馈感知 |
| T1.4 | 打开相机 | 入口识别时间 |
| T1.5 | 添加常用语 | 流程完成率 |
| T2 | 紧急功能定位 | 定位速度 |
| T3 | 长时间阅读 | 疲劳评分、理解度 |
| E1 | 错误恢复 | 恢复时间、成功率 |

### 4. 评估工具

- **NASA-TLX**：认知负荷评估（6维度）
- **SUS**：系统可用性量表（10题）
- **功能需求问卷**：28项无障碍功能重要性评估
- **A/B偏好测试**：5维度偏好对比

### 5. 数据分析

**定量分析**：
- 描述性统计（均值、标准差、中位数）
- 重复测量方差分析（2×4混合设计）
- 效应量计算（Cohen's d）
- 相关分析

**定性分析**：
- 主题分析（6阶段流程）
- 编码框架（视觉、操作、认知、情感、辅助技术）
- 定性-定量整合

**可视化**：
- 任务完成时间对比图
- NASA-TLX雷达图
- SUS得分分布图
- 功能需求热力图

## 快速开始

### 实验准备

1. **阅读实验方案**：详细阅读 `实验方案.md` 了解整体设计
2. **准备材料**：打印或使用电子版实验材料
3. **环境检查**：确保测试设备、录屏软件就绪
4. **预测试**：进行2人预测试，调整流程

### 执行测试

1. **招募参与者**：通过残联、老年大学、听障协会等渠道
2. **签署同意书**：使用 `实验材料/知情同意书.md`
3. **执行测试**：按照 `测试脚本.md` 逐步进行
4. **记录数据**：使用标准化数据记录表

### 数据分析

1. **数据录入**：将纸质问卷录入电子表格
2. **质量检查**：检查缺失值、异常值、逻辑一致性
3. **统计分析**：按照 `数据分析方案.md` 进行分析
4. **生成报告**：整合定量和定性发现

## 关键发现预期

### 假设1：改进设计显著提升任务成功率
- **H1**：改进设计的任务成功率显著高于现有设计
- **测量**：各任务成功率对比
- **预期**：平均提升15-25%

### 假设2：改进设计降低认知负荷
- **H2**：改进设计的NASA-TLX得分显著低于现有设计
- **测量**：6维度认知负荷评分
- **预期**：平均降低20-30%

### 假设3：改进设计提升用户满意度
- **H3**：改进设计的SUS得分显著高于现有设计
- **测量**：SUS总分
- **预期**：从"边缘"提升至"可用"或"良好"

### 假设4：不同用户组需求存在差异
- **H4**：各用户组对无障碍功能的需求优先级不同
- **测量**：功能需求问卷
- **预期**：低视力组重视视觉辅助，听障组重视触觉辅助

## 结果应用

### 优先级决策矩阵

| 优先级 | 改进项 | 依据 |
|--------|--------|------|
| P0 | 高对比度模式 | 影响广、实施易、WCAG必须 |
| P0 | 字体缩放适配 | 影响广、WCAG必须 |
| P0 | 目标尺寸增大 | 影响广、实施易、WCAG AA |
| P1 | 屏幕阅读器优化 | WCAG必须、听障用户需求 |
| P1 | 触觉反馈 | 听障用户高需求、实施易 |
| P1 | 减少动画选项 | WCAG必须、部分用户必需 |
| P2 | 简化界面模式 | 老年用户需求、实施复杂 |

### 设计迭代流程

```
实验结果 → 问题识别 → 优先级排序 → 设计改进
                                              ↓
全量发布 ← 数据监控 ← 小范围发布 ← 内部测试
```
