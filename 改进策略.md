# 《无声世界，指语成声》双向互动微信小程序 UI/UX 落地全案策略

## 一、 核心设计语言 (Design System)

我们摒弃传统的“平铺式”页面，采用**“原子化、医疗级极简、温暖科技感”**的设计语言，确保两种人群在使用时都感到安全、高效、高级。

### 全局主色调 (Palette)
- **基础背景 (Background)**：#F8FAFF（极淡的蓝白色，护眼且具空间感）
- **卡片层 (Surface)**：#FFFFFF（纯白，配合背景形成微弱的层级差）
- **品牌主色 (Primary)**：#007AFF（清爽科技蓝，用于主按钮、加载动画）
- **辅助色 (Secondary)**：#E3F2FD（浅蓝，用于选中状态、背景点缀）

### 形状与阴影 (Shape & Shadow)
- 所有卡片与弹窗一律采用 **32rpx 大圆角**。
- 采用**弥散阴影 (Diffuse Shadow)** `rgba(0, 122, 255, 0.08)`，取代生硬的边框线。

### 触觉反馈 (Haptics)
在关键交互（模式切换、3D模型加载完成、语音播报）时，统一调用微信原生的  
`wx.vibrateShort({type: 'light'})` 轻震动，提升高级感。

## 二、 全局核心：顶部“灵动胶囊”切换器

**目标**：抛弃繁琐的底部 TabBar 切换，实现零层级、无缝的身份转换。

### 视觉呈现
- 位于页面顶部居中（状态栏下方），纯白色半透明毛玻璃材质的长条胶囊。
- 左侧文字“健听人”，右侧文字“听障人士”。

### 交互动作
- 当前选中的模式底色为浅蓝色（#E3F2FD），文字加粗。
- 点击切换时，浅蓝色高亮块具有**弹性阻尼感 (Spring Animation)**，像果冻一样滑向另一侧。
- 切换瞬间，页面下方的内容模块进行极快的 Opacity 0 → 1 的淡入淡出（不发生页面白屏跳转）。

## 三、 用户首次打开：沉浸式引导 (Onboarding)

**目标**：建立共情，并让用户自选身份，记录偏好。

1. **第一屏 (共情)**：白色背景，中心为蓝色线框插画（手语与文字的交汇）。文案：“打破无声的隔阂”。
2. **第二屏 (功能)**：展示 3D SMPL 模型的线框缩影。文案：“将文字转化为灵动的手语”。
3. **第三屏 (选择)**：屏幕下方出现两个巨大的白色按钮（带浅蓝色阴影）：
   - **[我是健听人]** → 点击后进入健听模式，设置 `wx.setStorageSync('userMode', 'normal')`
   - **[我是听障人士]** → 点击后进入听障模式，设置 `wx.setStorageSync('userMode', 'deaf')`

## 四、 健听人模式首页：“翻译舱”

**目标**：极其克制，视觉中心全部留给“输入文字转手语”的核心诉求。

### 页面布局
- **主模块（屏幕上半部分）**：一个占据屏幕 40% 的巨大纯白卡片（内凹感）。  
  内有极简的文字提示：“输入文字或说话，转换为 3D 手语...”。  
  右下角悬浮一个浅蓝色的麦克风图标，支持语音直达。
- **次级模块（屏幕下半部分）**：
  - 横向卡片 A：“识别对方手语”（点击调用摄像头，背景为模糊的实景）。
  - 横向卡片 B：“手语科普与学习”（原有功能折叠至此）。

**交互流**：点击主卡片输入文字，点击右下角“生成”按钮 → 立即触发全屏沉浸视窗。

## 五、 核心高光：全屏 SMPL 3D 沉浸视窗

**目标**：用惊艳的视觉和优雅的加载，掩盖后端模型转换的时间（不采用分段回传的补偿方案）。

### 1. 入场与加载态 (Loading State)
- **动作**：点击生成后，首页背景瞬间变为 #F8FAFF（浅蓝白），首页输入框平滑上移缩小至顶部，屏幕从下往上弹出全屏视窗。
- **视觉**：屏幕正中央出现“水滴波纹脉冲”动画（三个浅蓝色的圆环如水波般缓缓向外扩散并淡出）。
- **文案**：下方显示一行字带有呼吸灯效果的文字：“AI 正在为您构建手语序列...”。

### 2. 展示态 (Showtime) - 数据返回后
- **环境渲染**：纯白色的虚拟实验室背景，底部有一个淡淡的浅蓝色圆形倒影（让模型有踩在地上的实感，而非悬浮）。
- **模型材质**：SMPL 3D 模型采用哑光浅蓝灰材质，不使用过于仿真的皮肤，强调工具属性和科技感。手指部分增加微弱的阴影对比，确保手势清晰。
- **文字对照**：在 3D 模型脚下，悬浮一个纯白色毛玻璃气泡，显示当前正在翻译的文字内容。
- **控制面板**：屏幕最底部提供极简的操作栏：[0.5x 慢放]、[重播]、[分享]。

### 3. 优雅退出 (Exit Gesture)
- 屏幕顶部居中有一个微小的浅灰色横条（Handle indicator）。
- 支持用户全局向下滑动手势。当向下拉动超过屏幕 1/5 时，3D 视窗伴随弹簧动画缩回底部，丝滑返回健听首页。

## 六、 听障人士模式首页：“表达矩阵”

**目标**：大色块、盲操作、即时发声。颜色暗示优先级。

### 页面布局
- **即时发声区（屏幕上半段）**：超大字号的文本输入框。输入完成后，右侧紧跟一个巨大的实心蓝色按钮 **[🔊 播放]**。点击立刻外放声音，同时按钮会出现蓝色波纹扩散特效，让听障人士“看”到声音正在播放。
- **快捷短语瀑布流（屏幕下半段）**：
  - 将常用语设计为圆角矩形贴片。
  - **色彩逻辑**：
    - 紧急/求助类（深蓝色块，白色文字）
    - 日常问候类（浅蓝色块，深蓝文字）
    - 业务办理类（纯白色块，蓝色文字）
  - 点击贴片，直接触发大喇叭语音播报。
- **底部悬浮**：右下角悬浮一个圆形的“相机”图标，点击进入录制自己手语转文字的界面。

## 七、 前后端 API 对接规范 (针对一次性加载)

为配合 UI 的“水滴脉冲加载”，前端与后端的交互时序必须如下严格执行：

1. **触发阶段**：前端显示脉冲动画 → 锁死返回按钮 → 发起 `wx.request`。
2. **后端处理**：后端接收文本，送入模型推理，生成完整的 SMPL 动作参数 (JSON 数组) 组合。
3. **返回阶段**：前端接收到完整的 JSON 数据 → 耗时 0.2s 初始化 Three.js/WebGL 画布 → 隐藏脉冲动画，伴随一次轻震动，渐显 3D 模型开始播放。
